# 项目健康度检查

## Step 1: 获取上下文

```
读取 .yunxiao.json 获取 organizationId、spaceId
调用 get_project 获取项目基本信息
调用 list_sprints(status: ["DOING"]) 获取当前进行中的迭代
```

## Step 2: 采集数据

```
全量工作项扫描：
  调用 search_workitems(category: "Req,Task,Bug", spaceId, perPage: 200)
  按需翻页获取全部数据

分维度统计：
  1. 状态分布：按 status 分组计数
  2. 逾期检测：planEndDate < 今天 且 status 非已完成
  3. 无人认领：assignedTo 为空 且 status 非已完成
  4. 瓶颈检测：同一状态停留 > 7 天（对比 updateStatusAt）
  5. 负载分布：按 assignedTo 分组统计进行中任务数
```

## Step 3: 生成报告

```
# 项目健康度报告 - {项目名}
> 生成时间：YYYY-MM-DD HH:mm

## 📊 总览
| 指标 | 数值 |
|------|------|
| 总工作项 | X |
| 进行中 | X |
| 已完成 | X |
| 逾期 | X（🔴 占比%） |
| 无人认领 | X |

## ⏰ 逾期项（按逾期天数降序）
| 编号 | 标题 | 负责人 | 计划完成 | 逾期天数 |

## 🚧 瓶颈项（同状态停留 > 7 天）
| 编号 | 标题 | 当前状态 | 停留天数 | 负责人 |

## 👤 负载分布
| 成员 | 进行中 | 待处理 | 建议 |
标记过载（> 5 项）和空闲（0 项）成员

## 🏥 健康度评分
综合评分：X/100
- 完成率得分：X
- 逾期得分：X（逾期越多扣分越多）
- 认领率得分：X
- 流转效率得分：X
```

## Step 4: 输出与跟进（可选）

```
方式一：输出 markdown 报告到本地
方式二：作为评论发布到云效项目
  调用 create_work_item_comment 附到指定工作项

标记需关注项：
  对逾期项 → 可批量添加评论提醒负责人
  对无人认领项 → 列出建议指派人选
```
